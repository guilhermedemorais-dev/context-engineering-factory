# Acceptance Criteria

## Objetivo
Definir criterios de aceite (DoA) para o framework e entregas.

## DoA por tipo

### Docs/Contexto
- Consistencia interna.
- Referencias corretas.
- Sem contradicoes.

### Bots/Runtime
- Executavel end-to-end.
- Logs estruturados.
- Escrita segura e paths validados.
- Dry-run quando aplicavel.

### CI/CD
- Gates definidos e alinhados com quality-bars.
- Templates coerentes com estrategia.

### Design-system
- Tokens aplicados.
- Acessibilidade minima atendida.
- Catalogo atualizado.

## Evidencias para PR (template)
- Contexto usado: [paths]
- Mudancas feitas: [resumo]
- Tests executados: [lista]
- Evidencia MCP/registry (quando aplicavel): [links]
- Gates atendidos: [sim/nao]

## Criterios de bloqueio (hard fail)
- Falha em gate critico.
- Contradicao de contexto.
- Evidencias ausentes.
