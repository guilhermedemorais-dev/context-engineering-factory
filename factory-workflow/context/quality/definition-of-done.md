# Definition of Done (DoD)

## Objetivo
Definir criterios minimos de conclusao por tipo de mudanca.

## DoD para contexto/docs
- Contexto lido e referencias atualizadas.
- Consistencia validada.
- Checklist preenchido em `factory-workflow/cicd/checklist.md`.

## DoD para runtime/bots
- Execucao minima validada.
- Logs e escrita segura confirmados.
- Testes aplicaveis executados.
- Checklist em `factory-workflow/context/codex/checklist-delivery.md` preenchido.

## DoD para factory-workflow/design-system/ui
- Tokens aplicados e catalogo atualizado.
- Acessibilidade minima validada.
- Evidencia de reuso quando aplicavel.

## DoD para factory-workflow/cicd/tests
- Gates coerentes com quality-bars.
- Jobs mapeados em `factory-workflow/cicd/templates.md`.
- Checklist em `factory-workflow/cicd/checklist.md` preenchido.

## Evidencias minimas
- Gates ok (`factory-workflow/cicd/gates.md`).
- Checklists preenchidos (`factory-workflow/context/codex/checklist-delivery.md`, `factory-workflow/cicd/checklist.md`).
