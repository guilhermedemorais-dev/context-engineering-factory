# Components

## Catalogo
Este arquivo define o modelo de documentacao para cada componente.

## Modelo por componente
- Proposito
- Variantes
- Estados (default/hover/focus/disabled/loading/error)
- Acessibilidade minima
- Tokens usados

## Regra principal
Antes de criar componente novo, buscar em registry conforme `factory-workflow/context/ui/component-policy.md`.

## Como registrar componente novo
1. Validar reuso em registry e design system.
2. Documentar no modelo acima.
3. Referenciar tokens usados.
4. Atualizar acessibilidade quando aplicavel.

## Checklist minimo
- [ ] Proposito claro.
- [ ] Estados definidos.
- [ ] Tokens listados.
- [ ] Acessibilidade minima definida.
